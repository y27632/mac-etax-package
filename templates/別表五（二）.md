## ① 法人税及び地方法人税（IEB00000）

### 前期以前の未払分（IEB00010）— 未払がある場合のみ記入

| タグID | 項目 | 値 |
|--------|------|----|
| IEB00020 | 事業年度（自） | |
| IEB00030 | 事業年度（至） | |
| IEB00050 | 期首現在未納税額 | |
| IEB00160 | 期末現在未納税額 | |

> 当期中に納付がない場合は IEB00050 = IEB00160。
> 複数期分ある場合は期間を合算して1行にまとめる（maxOccurs=2なので最大2行）。
> 第1期の場合は前期以前がないため省略。

### 当期分

| タグID | 項目 | 値 |
|--------|------|----|
| IEB00320 | 確定法人税額＋確定地方法人税額（円） | |
| IEB00350 | 損金経理による納付額（円） | |

> IEB00320: 別表一 BGB00210 ＋ BGC00140。
> IEB00350: 当期に損金処理で納付した場合のみ。

### 計（IEB00360）

| タグID | 項目 | 値 |
|--------|------|----|
| IEB00380 | 期首現在未納税額 合計 | |
| IEB00390 | 当期確定発生額 | |
| IEB00520 | 期末現在未納税額 | |

> IEB00380: 前期以前行 IEB00050 の合計（前期以前がなければ0）。
> IEB00390: IEB00320と同値。
> IEB00520: IEB00380 ＋ IEB00390。

## XTX生成プロンプト（AIへ）

上記の記入欄に入力された値をもとに、別表五(二)（HOA522）のXML断片をPython辞書形式で出力してください。

### 基本ルール

- バージョンは frontmatter の `xtx_version` を使用する
- 前期以前の行（IEB00010/IEC00010）は未払がある場合のみ出力する
- 計（IEB00360/IEC00490/IED00360）は値が0でも省略しない
- REFタグは文字列でそのまま出力する

### 計算順序

1. `IEB00390` = IEB00320（確定法人税額 + 確定地方法人税額）
2. `IEB00520` = IEB00380 ＋ IEB00390
3. IEC: 同様の計算を道府県民税で実施
4. `IEE00160` = IEE00040 ＋ IEE00050（事業税がある場合）

### 注意事項

- 第1期: 前期以前行（IEB00010/IEC00010）を省略し、IEB00380/IEC00510=0
- 市町村民税（IED）は eLTAX で申告するため、e-Tax上は0で必ず出力する
- IEB00010/IEC00010 の maxOccurs=2。複数期分は事業年度を合算して1〜2行に収める
